CK package manager unifies installation of code, data and models across different platforms and operating systems using the most appropriate solutions including wget, make, cmake, scons, spack, vcpkg, conan, easybuild, etc.

You can install a CK package into CK virtual environment as follows: 

 ck pull repo:{Repo UOA - see below}
 ck install package:{Package UOA - see below}
using tags: 
 ck install package --tags={some tags from below}
to an unusual path: 
 ck install package:{Package UOA - see below} --install_path={full installation path}
to a CK virtual environment entry : 
 $ ck set kernel var.install_to_env=yes
 ck install package:{Package UOA - see below}
or for a different OS target (Android): 
 ck ls os:android* | sort
 ck install package:{Package UOA - see below} --target_os={OS UOA from above}
You can then see or use registered virtual CK environments for installed packages as follows: 
 ck show env
 ck show env --tags={some tags from below}

 ck virtual env:{UID from above}
 ck virtual env --tags={some tags from below}

See 

ck install package --help
for more installation options. See related CK soft detection plugins, CK documentation, "how to contribute" guide, ACM ReQuEST-ASPLOS'18 report and the latest CK paper for further details. See ck-spack repository connecting CK and spack package manager.

You can obtain repository with a given program (workflow) as follows: 
 ck pull repo:{Repo UOA - see below}
You can then compile this program if necessary (CK will automatically detect software dependencies or install missing packages) as follows: 
 ck compile program:{Program UOA - see below}
You can run this program as follows: 
 ck run program:{Program UOA - see below}
You can execute program pipeline which chains together various plugins to set up and monitor frequency, prepare data sets, compile program and run it several times, etc: 
 ck pipeline program:{Program UOA - see below} ...
You can check extra options to customize above pipeline as follows: 
 ck pipeline program:{Program UOA - see below} --help
You can customize pipeline execution via variables: 
 ck pipeline program:{Program UOA - see below} --env.VAR1=PARAM1 ...
You can also benchmark a given program which includes executing this program several times, performing statistical analysis, unifying I/O, etc: 
 ck benchmark program:{Program UOA - see below}

See CK documentation, "how to contribute" guide, and the latest CK paper for further details.
